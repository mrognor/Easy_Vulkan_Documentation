97  // Second paramert is shader type
98  // Third parametr is optional and set shader entry point.
99  // That means that it's possible to combine multiple fragment shaders into a
100  // single shader module and use different entry points to differentiate between their behaviors.
101  // Fourth parametr is optional and set shader specialization.
102  // It allows you to specify values for shader constants.
103  // You can use a single shader module where its behavior can be configured at pipeline creation
104  // by specifying different values for the constants used in it. This is more efficient than
105  // configuring the shader using variables at render time, because the compiler can do optimizations
